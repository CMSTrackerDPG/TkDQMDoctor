Pixel Helper

This page contains the list of checks to be done for every run, but also some useful information about possible issues and links to the documentation.

Pixel check list

(TO BE UPDATED WITH THE NEW SUMMARY MAPS FOR 2018)
  • Check the number of cluster plot and the trends. Make sure the Pixel detector is giving data, in the case the trend present some features (i.e dips or peaks) compare with the corresponding histogram in the Strip, if the feature is present also there this mean it is due to a change in trigger rate or to a luminosity scan.
  • Check the on-track cluster charge and size comparing against a suitable reference in particular check if the distributions match and the MPV value is stable. In case you find a distorted shape (i.e multiple peaks are visible), check with the help of the ROC maps if those clusters become from a particular region of the detector. Also the cluster charge for inner and outer ladder (one plot per layer) have to be checked against the reference.
  • Compare occupancy maps with a reference and to the previous run and write an elog in case of new dead FULL MODULES (in a way to avoid calls for stuck TBMs which are anyway taken into account in the Summary plot)
  • Compare the Digi and FED0f25 maps, and if there are holes in the Digi maps which are not showing up in the FEDof25 map ( (top)PixelPhase1/FED/DeadChannels per ROC), write an elog
  • Check the dead ROC number, if there is a difference with respect to the previous run (do not use the reference for this!), if so check the digi occupancy maps at ROC level and find the new holes (to be added to the shift report). The Pixel detector is fundamental for tracking, a new hole can cause issues in the reconstruction. If a large area is dead, inform the SL and submit an ELog with title: "New hole(s) in the Pixel detector" attaching an image showing clearly where the new hole appear.
  • If there are no new holes and the charge distributions are fine, the Pixel detector most probably is in a good shape.
  • You can then check the Error plot to see if there aren't issues in the readout. Finally check the hit efficiency and the residuals to make sure that the tracking is behaving.
