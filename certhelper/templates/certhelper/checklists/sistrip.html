SiStrip Helper

SiStrip check-list

  • Check general distributions: *# of Cluster trend*, *!OnTrackCluster (!StoN)*, *FED-Detected Errors and Summary*. You now have a "big picture" of the basic quantities: you know if the detector produce clusters, if there are tracks reconstructed out of the clusters and if the read-out Errors are in a reasonable range
  • Check the S/N of on-track clusters against the reference, it tells you if the local reconstruction is correct
  • Check the *!StripClusVsPixClus* for stability (see the description for details), if in cosmics, check also !DeltaBx_vs_ApvCycle
  • At this point the !TkMaps should be ready to inspect, start looking at them. At first look to genral variables !TkMaps *Number of Clusters per module*, *Number of clusters on-track per module*, *Mean value for S/N for on-track cluster corrected for the angle*, *Cluster charge / cm from track* checking if there are new "hot" regions (see the plot description for details)
  • Check the *!FEDChannelErrors* !TkMap, in the case new "coloured" regions appear, check in the GUI the *Any Error Per APV Per Event* plot to identify what is the FED and the APV range giving errors (see description for details)
  • Finally, go back to the *Report Summary Map*, open the *!QTestAlarms* map and follow Guidelines to identify bad components or single modules in general
  • Before the end of the shift, check the *HDQM trends* including all the runs taken during the week (for cosmics, collisions in Express and Prompt)

Tips for investigating potential issues

Descriptions of SiStrip plots

A list of plots to check follows, with a link to the their description (along with the actions to be taken), note that only few plots are relevant for the certification process, the other plots in the list are meant for futher investigation.
  1. SiStrip Report Summary Map description NOTE: this plot is fundamental, and is the starting point to monitor the bad components evolution on a run by run basis (see the Guidelines to identify bad components or single modules in general)
  2. # of Cluster trend description
  3. FED-Detected Errors, and FED-Detected Errors Summary description
  4. (in case the FED error histograms show problems) Any Error Per APV Per Event description
  5. OnTrackCluster (StoN) description
  6. StripClusVsPixClus description (not for Cosmics)
  7. Only for long ( > 100 LS) Cosmic runs: DeltaBx_vs_ApvCycle description
