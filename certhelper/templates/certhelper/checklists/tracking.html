Tracking Checklist

Tracking Plots:

Plots in QuickCollection:

  1. the Report Summary Map description
  2. track parameters (#Tracks, Pt, Eta, Phi, Chi2, #hits) description

Tracker Maps

Other Plots:
  1. the Tracker Maps which have to be produced by offline shifter right after checking the plots in the GUI (see instructions):
    • Valid, Missing, Inactive hits description

HDQM trends

  1. historic DQM trend plots: average number of rec-hits per track, average pT, average chi2/ndof description

Suggested sequence

1 Check GUI plots (Tracking plots are quite a few, in the GUI they are all located in the same folder) 1 Check TkMaps

Before the end of the shift, check the HDQM trends including all the runs taken during the week (for cosmics, collisions in Express and Prompt)

Identify bad components

Finally, having inspected all the plots, and hence having the "big picture" of data quality, it is time to investigate on bad components:

  • are there any new BAD component with respect the previous run in the same conditions?
  • If yes, what is the reason to be flagged as bad?
  • did we recover some "known" bad component?
