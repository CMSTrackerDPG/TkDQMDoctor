Tracking checks

  • Check GUI plots (Tracking plots are quite a few, in the GUI they are all located in the same folder)
  • Check TkMaps
  • Before the end of the shift, check the HDQM trends including all the runs taken during the week (for cosmics, collisions in Express and Prompt)
Finally, having inspected all the plots, and hence having the "big picture" of data quality, it is time to investigate on bad components:
  • are there any new BAD component with respect the previous run in the same conditions?
  • If yes, what is the reason to be flagged as bad?
  • did we recover some "known" bad component?

Tips for investigating potential issues

Description of the Tracking Plots:

  • the Report Summary Map description
  • track parameters (#Tracks, Pt, Eta, Phi, Chi2, #hits) description
  • the Tracker Maps which have to be produced by offline shifter right after checking the plots in the GUI (see instructions): Valid, Missing, Inactive hits description
  • istoric !DQM trend plots: average number of rec-hits per track, average pT, average chi2/ndof description
